# C4 AI Image Lab — 발표 자료

---

## 1. 개발에 가장 중점을 둔 사항

**"백엔드 없이 브라우저 하나로 동작하는 것"** 이었습니다.

초기엔 FastAPI + SQLite 백엔드를 구축했습니다. 이미지를 DB에 저장하고, 히스토리를 불러오는 구조였습니다. 그런데 개발하면서 "왜 서버가 필요한가?"를 계속 물었고, 결국 이미지는 AI API가 만들어서 돌려주면 끝이라는 결론에 도달했습니다. DB도, 서버도 필요 없었습니다.

그 결과 **서버 비용 0원, 인프라 관리 없음, Vercel로 5분 배포** 라는 구조가 됐습니다.

## 2. 어떤 API를 사용했고, 왜 사용했는지, 과금 체계는

### 최종 선택: Stability AI Core

| 항목 | 내용 |
|------|------|
| 서비스 | [Stability AI](https://platform.stability.ai) |
| 모델 | Stable Image Core |
| 과금 | 이미지 1장당 크레딧 차감, 가입 시 25크레딧 무료 |
| 장점 | 고품질, 한국어 프롬프트 지원, 안정적인 응답 |

### API를 바꿔온 과정 (삽질 기록)

개발 과정에서 API를 **총 4번** 변경했습니다.

| 순서 | 서비스 | 교체 이유 |
|------|--------|----------|
| 1차 | **Pollinations.ai** | 무료, 키 불필요 → 새 이미지 생성 시 지속적으로 500 에러 발생. 캐시된 이미지만 응답하는 서버 장애 |
| 2차 | **fal.ai SDXL-Lightning** | 빠르고 저렴 ($0.00055/장) → 무료 크레딧 $10 소진됨 |
| 3차 | **Together AI** | fal.ai 대안으로 시도 → 무료 크레딧 없음, 즉시 과금 |
| 4차 | **Hugging Face** | 무료 인퍼런스 API 시도 → API 엔드포인트가 410 Gone (서비스 종료됨) |
| **최종** | **Stability AI** | 가입 시 25크레딧 무료, API 안정적, 한국어 지원 ✅ |

이 과정이 오히려 **각 AI API 서비스의 특성과 한계를 직접 몸으로 배운 경험**이 됐습니다.

## 4. 개발에 어려웠거나 고민하다가 건너뛴 부분

### 어려웠던 것

**채팅 스크롤 버그**
가장 시간을 많이 쏟은 문제였습니다. 이미지가 생성되면 화면이 스크롤되지 않고 이미지가 상단에 고정되는 현상이 발생했습니다. CSS를 세 군데 수정하고, `scrollIntoView` → 컨테이너 직접 제어 방식으로 바꾸고 나서야 해결됐습니다. 원인은 Flexbox 레이아웃에서 `min-height: 0`이 빠진 것이었습니다. 아주 사소한 한 줄이 꽤 긴 디버깅으로 이어졌습니다.

**API 불안정성**
Pollinations.ai가 캐시된 요청만 응답하고 새로운 생성 요청에 500을 반환하는 상황을 만났습니다. 처음엔 코드 문제라 생각해 계속 수정했지만, 알고 보니 서버 측 장애였습니다. 외부 서비스 의존도가 높을수록 서비스 안정성이 내 코드 밖에서 결정된다는 걸 체감했습니다.

### 고민하다가 건너뛴 것

**히스토리 기능**
DB 저장 → 사이드바 히스토리 표시까지 완성했다가 전부 걷어냈습니다. 서버 없이 동작하는 구조로 전환하면서 필요가 없어졌고, 로컬스토리지 대안도 있지만 "지금 당장 필요한가?"를 물었을 때 아니라고 판단해 건너뛰었습니다.

## 5. 업그레이드하고 싶은 부분

우선순위 순으로:

1. **히스토리 유지** — 로컬스토리지에 저장해서 새로고침해도 이전 생성 기록 보존. 지금은 새로고침하면 다 사라집니다.

2. **이미지 다운로드 버튼** — 생성된 이미지를 바로 저장할 수 있어야 실용적입니다.

3. **이미지 비율 선택** — 현재 정사각형(1:1) 고정입니다. 16:9, 9:16 선택 옵션을 추가하면 SNS 콘텐츠 제작 등 활용도가 높아집니다.

4. **한국어 프롬프트 자동 번역** — Stability AI는 한국어를 지원하지만, 영어 프롬프트가 더 정확한 결과를 냅니다. Claude API로 프롬프트를 영어로 자동 변환하면 품질이 향상됩니다.

5. **Vercel → 커스텀 도메인 연결** — 현재 vercel.app 주소입니다. 도메인을 연결하면 서비스처럼 보입니다.
